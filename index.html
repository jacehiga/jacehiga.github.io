NBA Betting Analysis and Projections

Stack: SQL • Grafana • Python • API Scraping • Railway




This data engineering project aimed to support betting analysis by projecting NBA player statistics. Using Python and the nba_api package, I built a daily scraper that collected player data and saved it in JSON format. The data was uploaded to Railway and transformed using SQL into structured tables for efficient querying. I analyzed trends in player performance such as recent scoring averages, minutes played, and matchup history, and visualized them through an interactive Grafana dashboard. These insights enabled more informed player prop bets by identifying favorable matchups and spotting deviations from sportsbook expectations. Through this project, I strengthened my skills in data pipeline automation, API integration, and dashboard design for real-time analytics.

MLB Data Acquisition

Stack: SQL • Python • Selenium • API Scraping • Railway



This project focuses on building an end-to-end pipeline to collect and structure MLB data for analysis. I developed two automated scrapers: one using Selenium to extract dynamic content from MLB.com, and another using the statsapi package for structured player and game statistics. The data is collected daily, stored in JSON format, and deployed via Railway. It is then ingested into a PostgreSQL database where it can be queried for trend analysis, matchup evaluations, and potential betting insights. Through this project, I gained experience integrating multiple data sources, handling dynamic web content, and designing scalable ingestion pipelines for structured sports analytics.

Datafest: Legal Data Analysis

Stack: R • tidyverse • Textual Analysis • tidytext • ggplot2



As part of a 48-hour DataFest competition, I led exploratory analysis on legal data provided by the American Bar Association using the tidyverse package in R. I applied tidytext to perform sentiment and keyword analysis, uncovering patterns in legal language and case framing. To strengthen our insights, I incorporated external datasets and conducted targeted research to contextualize trends. I visualized these findings using ggplot2, producing clear and compelling graphics that helped communicate our story to judges. Our analysis helped the American Bar Association identify where pro bono legal support is most needed, based on geographic and case-type trends. This project deepened my skills in textual analysis, rapid data exploration, and collaborative problem-solving under time pressure.

Spelling Bee Word Generator

Stack: JavaScript • Observable


This project involves the development of a solver tool for The New York Times “Spelling Bee” game. Users input a set of letters, and the tool generates a table of valid words, sorted by length with the longest words prioritized. The output also includes a dynamic word cloud, where words longer than seven letters are highlighted in bold and pangrams (words that use all available letters) are distinguished in teal for quick recognition.
